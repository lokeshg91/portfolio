# design-process
How a project goes from brief to shipped.
Lokesh Gurjar11:05 AM

Every project is different, but the shape is usually the same:

1
Discover
Understand business goals, users, competitors, and project requirements.
2
Define
Map user journeys, information architecture, and product strategy.
3
Design
Create wireframes, high-fidelity interfaces, components, and design systems.
4
Prototype
Build interactive prototypes for validation with users and stakeholders.
5
Deliver
Developer-ready specs, assets, documentation, and implementation support.
